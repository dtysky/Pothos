# Research

这里存放不隶属于大型项目的单次调研、行前案头研究和专题资料整理。

它适合承接一次明确的问题、旅程、地点、作品或主题，但不默认沉淀为长期知识库，也不默认升级为一个完整创作项目。

## 使用场景

- 某次旅行或拍摄前的文学、历史、地理、社会背景调研
- 某个主题的临时资料包，例如一座城市、一条路线、一组作品
- 尚未确定是否会发展成大型项目的前期研究
- 一次对话中形成的可复用报告，但还不适合写入 `.pothos/knowledge/`

## 与其他目录的边界

- `.pothos/knowledge/`：长期稳定、后续反复调用的人物、文风和项目方法论。
- `projects/*/research/`：已经归属某个大型项目的研究资料。
- `notes/`：咨询过程、判断记录、阶段性讨论归档。
- `research/`：独立的单次调研报告和专题案头资料。

## 命名规则

优先使用：

```text
YYYYMMDD-topic-scope.md
```

示例：

```text
20260617-hebei-wanqing-taihang-industrial.md
```

如果调研没有明确日期，可以使用：

```text
topic-scope.md
```

## 写作要求

- 明确区分事实、推断和建议。
- 外部资料需要保留来源链接或出处。
- 避免堆砌摘录，优先写成可直接用于创作判断的结论。
- 如果后续发展为大型项目，应迁移到 `projects/<project-code>/research/`，并在这里保留迁移说明或索引。
