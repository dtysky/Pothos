# Shot Design

## 目的

为影像类项目设计镜头时，确保每个镜头描述同时承载叙事意图和可执行的拍摄信息。输出必须让 Pharos 能直接消费为机位规划，无需二次翻译。

## 何时使用

当用户提出以下需求时使用：

- 设计纪录片、短片、影像作品的镜头
- 为某个场景或段落规划视觉表达
- 需要把叙事意图落成具体的拍摄方案

## 必读资料

- 当前项目的 `design/concept.md`（理解叙事意图）
- 当前项目的 `design/structure.md`（理解段落位置）

## 写作规范

每个镜头描述必须包含三层信息，用自然语言写，不使用结构化数据格式，但信息必须齐全。

### 时空层

- **地点**：具体到可定位的程度（城市 + 具体位置）
- **时间窗口**：拍摄时段（日出前30分钟、蓝调时刻、正午等）
- **朝向**：镜头面向的方向或关键参照物
- **季节/天气偏好**：如有要求

### 镜头层

- **景别**：远景 / 全景 / 中景 / 近景 / 特写
- **运动**：固定 / 缓推 / 横移 / 跟随 / 手持
- **时长**：预估持续时间（拍摄素材时长 + 成片预估时长）
- **特殊技法**：延时、升格、多机位等
- **设备**：机身、镜头、稳定器、三脚架、滤镜等关键器材

### 叙事层

- **内容**：画面里有什么
- **意图**：这个镜头在作品中承担什么功能
- **旁白/声音**：如有配合的文字或环境声设计

## 工作流

### Step 1

先从叙事出发：这个镜头要表达什么？它在整体结构中的位置是什么？

### Step 2

把叙事意图落成具体画面：画面里有什么，怎么拍能让意图成立。

### Step 3

补全时空和设备信息：地点精确到可定位，时间精确到可规划，设备精确到可执行。

### Step 4

检查三层信息是否齐全。如果某项确实不确定，标注"待定"并说明原因，不要省略字段。

## 示例

```
## 朝天门 · 两江交汇

- 地点：重庆，朝天门码头，嘉陵江与长江交汇处
- 时间：日出前20分钟至日出后40分钟
- 朝向：面朝东偏南，对准两江交汇的水面分界线
- 天气：晴或薄雾均可，大雾备选方案

- 景别：远景，逐渐推至全景
- 运动：三脚架固定，缓推
- 时长：拍摄素材约20分钟，成片预计30-45秒
- 技法：延时，间隔3秒
- 设备：A7M4 + 24-70 GM II，三脚架，CPL

- 内容：两条不同颜色的江水在码头前交汇，晨光逐渐照亮对岸
- 意图：两条河流的汇合作为"寻找源头"的反向隐喻——他从汇合处出发，逆流回到分叉之前
- 声音：环境声为主（江水、远处船笛），无旁白
```

## 与 Pharos 的关系

Pothos 负责叙事意图和镜头设计，Pharos 负责拍摄执行。本 skill 的输出格式已确保 Pharos 可直接读取时空层和镜头层信息，用于行程规划和机位安排。
